# 计算机图形学实验七：质点弹簧模型

## 一、实验信息

- 实验名称：质点弹簧模型
- 授课教师：张鸿文
- 助教：张怡冉
- 课程主页：https://zhanghongwen.cn/cg

## 二、实验目标

本实验使用 Taichi 构建一个可交互的三维布料模拟程序。布料被离散为 `20 x 20` 的质点网格，相邻质点通过弹簧连接，并在重力、阻尼、弹力和碰撞约束下实时运动。

实验主要完成以下目标：

- 掌握动态场景渲染：使用 Taichi GGUI 创建 3D 场景和交互控制面板。
- 理解质点-弹簧模型：实现结构弹簧、阻尼力、速度钳制等基础物理模拟模块。
- 对比数值积分方法：实现显式欧拉、半隐式欧拉和定点迭代近似的隐式欧拉。
- 理解 GPU 编程基础：使用 `@ti.kernel` 和 `@ti.func` 组织并行计算，并注意初始化阶段的同步顺序。
- 完成选做内容：增加剪切弹簧、弯曲弹簧和球体碰撞处理。

## 五、场景初始化

程序将布料初始化为一个位于空间上方的正方形网格：

- 网格大小：`20 x 20`
- 质点数量：`400`
- 初始宽度：`3.2`
- 顶部部分质点固定，用于模拟悬挂布料
- 初始速度和受力均为零

主要固定参数如下：

| 参数 | 数值 | 说明 |
| --- | --- | --- |
| `MASS` | `1.0` | 质点质量 |
| `DT` | `5e-4` | 单次物理积分时间步 |
| `SPRING_STIFFNESS` | `10000.0` | 弹簧劲度系数 |
| `MAX_VELOCITY` | `50.0` | 速度钳制上限 |
| `PHYSICS_SUBSTEPS` | `20` | 每帧物理子步数 |
| `IMPLICIT_ITERATIONS` | `5` | 隐式欧拉定点迭代次数 |

为了满足 GPU 同步要求，初始化被拆分为三个独立 kernel，并在 Python 侧顺序调用：

- `init_positions()`：初始化质点位置、速度、受力和固定点标记。
- `init_springs()`：初始化弹簧拓扑和弹簧原长。
- `init_render_indices()`：初始化布料三角面片索引。

这种拆分可以避免在同一个并行 kernel 中依赖尚未完成写入的状态，使初始化流程更清晰稳定。

## 六、质点-弹簧模型实现

### 1. 受力计算

程序将每个质点的受力分为三部分：

```text
F = F_gravity + F_damping + F_spring
```

其中重力为常量：

```text
F_gravity = m * g
```

全局速度阻尼用于消耗系统能量：

```text
F_damping = -k_d * v
```

弹簧力基于胡克定律。对于弹簧两端质点 `a` 和 `b`：

```text
F_a = -k_s (|x_a - x_b| - l) normalize(x_a - x_b)
```

程序在 `compute_forces_on()` 中计算每根弹簧的弹力和沿弹簧方向的相对速度阻尼。由于一根弹簧会同时写入两个质点的受力，而多个弹簧可能并行写入同一个质点，因此使用 `ti.atomic_add` 累加受力，避免多线程写冲突。

### 2. 速度钳制

显式欧拉在刚度较大或时间步较大时容易产生数值爆炸。程序实现了 `clamp_velocity()`：

```text
if |v| > v_max:
    v = normalize(v) * v_max
```

这样可以限制异常速度，避免布料在不稳定参数下瞬间飞出场景。

### 3. `@ti.func` 内联

以下函数均声明为 `@ti.func`：

- `compute_forces_on()`
- `clamp_velocity()`
- `collide_with_ball()`
- `spring_enabled()`

`@ti.func` 会在编译期内联到调用它的 kernel 中，减少函数调用开销，也便于将受力计算和积分更新合并在同一次 kernel 启动中完成。

## 七、三种积分方法

### 1. 显式欧拉

对应 kernel：`step_explicit()`

显式欧拉先使用旧速度更新位置，再使用当前受力更新速度：

```text
x_{t+1} = x_t + v_t * dt
v_{t+1} = v_t + a_t * dt
```

该方法实现简单，但在弹簧刚度较高时最容易发散。实验中固定使用较小的 `DT = 5e-4` 并配合速度钳制，避免显式欧拉在对比演示中过早数值爆炸。

### 2. 半隐式欧拉

对应 kernel：`step_semi_implicit()`

半隐式欧拉先更新速度，再用新速度更新位置：

```text
v_{t+1} = v_t + a_t * dt
x_{t+1} = x_t + v_{t+1} * dt
```

相比显式欧拉，它对弹簧系统更加稳定，也是本程序默认使用的积分方法。

### 3. 隐式欧拉

对应 kernel：`step_implicit_iter()`

隐式欧拉使用未来时刻的状态计算受力：

```text
v_{t+1} = v_t + a_{t+1} * dt
x_{t+1} = x_t + v_{t+1} * dt
```

完整隐式求解需要解非线性方程组。实验中使用定点迭代近似：先预测未来位置和速度，再反复基于预测状态计算受力并更新预测结果。为了适配 Taichi kernel 的并行循环限制，代码使用固定上限静态展开，并通过 UI 中的 `Implicit Iterations` 控制实际迭代次数。

## 八、GGUI 交互

程序使用 `ti.ui.Window`、`ti.ui.Scene` 和 `window.get_gui()` 构建 3D 场景与交互面板。

控制面板包含：

## 九、选做内容实现

### 1. 剪切弹簧

剪切弹簧连接网格单元的两条对角线：

```text
(i, j) -> (i + 1, j + 1)
(i, j) -> (i + 1, j - 1)
```

开启剪切弹簧后，布料对斜向形变的抵抗更强，网格不容易被拉成菱形。

### 2. 弯曲弹簧

弯曲弹簧连接间隔两个网格点的水平和竖直质点：

```text
(i, j) -> (i, j + 2)
(i, j) -> (i + 2, j)
```

开启弯曲弹簧后，布料会更倾向于保持整体平滑，局部折叠和尖锐弯曲会减少。

### 3. 球体碰撞

场景中放置了一个橙色球体，位置和半径为：

```text
center = (0.0, -1.15, 0.0)
radius = 0.55
```

每次积分后，程序检测质点到球心的距离。如果质点进入球体内部，就将其投影回球面，并去除指向球体内部的法向速度分量：

```text
if |x - center| < radius:
    x = center + normalize(x - center) * radius
    if dot(v, normal) < 0:
        v = v - dot(v, normal) * normal
```

这样可以得到简单稳定的空间碰撞效果。

## 十一、遇到的问题与解决方法

1. 多线程受力累加冲突

   问题：多个弹簧可能同时向同一个质点写入受力。

   解决：弹簧受力使用 `ti.atomic_add` 累加到两端质点。

2. 显式欧拉数值爆炸

   问题：刚度或时间步较大时，显式欧拉会快速发散。

   解决：增加 `clamp_velocity()` 速度钳制，并使用固定物理子步配合阻尼参数控制稳定性。

3. 初始化阶段状态依赖

   问题：弹簧原长依赖已经初始化好的质点位置。

   解决：将位置初始化、弹簧初始化和渲染索引初始化拆分为独立 kernel，并在 Python 侧顺序调用。

4. 隐式迭代中的 kernel 循环限制

   问题：Taichi 不允许在动态循环内部嵌套并行 `struct_for`。

   解决：使用静态展开的固定最大迭代次数，并在每一轮内部判断是否小于 UI 设定的实际迭代次数。

## 十二、实验总结

本实验完成了一个基于 Taichi 的实时质点-弹簧布料模拟程序。必做部分实现了结构弹簧、重力、阻尼、速度钳制、三种积分方法和 GGUI 交互控制；选做部分进一步加入剪切弹簧、弯曲弹簧和球体碰撞。

通过对比三种积分器可以观察到，显式欧拉实现最简单但稳定性最弱，半隐式欧拉在实时物理模拟中更加实用，隐式欧拉通过未来状态估计可以进一步提高稳定性。剪切弹簧和弯曲弹簧改善了布料的形变质量，球体碰撞则让动态场景更完整，也更容易观察布料与空间物体的交互。
